Tool Library CLI - CAB301

CAB301 was a more advanced version of CAB201, in that a sorting algorithm was to be implemented along with a binary tree.
Binary trees and the difference between a few sorting algorithms (bubble sort, merge sort, etc)

The main component of this project was the sorting algorithm used. I had chose bubble sort because it's fun, simple and effective. For such a use case like the one provided, a fast algorithm wasn't needed
We were given interfaces, of which were not straight forward and we were told later into the semester to ignore them.
Outside of that, the only real error I had made was not properly using the Tool Library System as intended and not splitting my code between classes properly.
The main differences from CAB201 was that I did not do an input loop, but had it integrated in the main loop, where each option would run a different method.

This project also gave me a sneak peak into what CAB401 would be about, where an empirical analysis was done showing the O notation efficiency of the sorting algorithm used. (O(n^2))
Oh I should mention the sorting algorithm was just to sort tables alphabetically or by value before printing (depending on user preference)

Similarly to CAB201, a more colourful CLI would have helped.
If it were allowed I would have also used a library for prettier table printing since these tables were occasionally larger than the window.
